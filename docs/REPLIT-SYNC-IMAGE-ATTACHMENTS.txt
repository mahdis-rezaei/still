WHAT GOES LIVE (since the last publish):
1. Image attachments (c8d542c): attach images to a page — upload, or "take a
   picture" on mobile — from BOTH the Today composer and the full entry view.
   - Bytes are app-encrypted (AES-256-GCM, same ENCRYPTION_KEY as the journal
     text) and stored in Object Storage; Postgres holds only metadata + the
     storage key. A stolen bucket is useless without ENCRYPTION_KEY.
   - Images are display-only: NEVER sent to the memory engine, NEVER resurfaced.
     The engine still reads plain-text `body` exactly as before.
   - New endpoints: POST /api/entries/:id/attachments (raw image, owner-scoped),
     GET /api/entries/:id/attachments (metadata list), GET /api/attachments/:id
     (streams the decrypted image, owner-only), DELETE /api/attachments/:id.
   - Account delete now also purges image objects; privacy export JSON gains an
     `attachments` array (metadata).

   "What keeps returning" / "Revisit a time" / tabbed memory home
