tiebreak DARK (off by default). You turn it on only after a live harness check.

WHAT THIS IS
A third soft selection tiebreak ("why today") in the engine's PASS2 prompt, in
the exact idiom of the existing CROSS-TIME and FRESH-GRIEF tiebreaks: it only
nudges between two candidates of COMPARABLE strength toward the one that
resonates with today (same season, an anniversary, or echoing the writer's
recent themes). emotional_center still decides any clear winner; the silence bar
is unchanged. It is gated behind an env flag and is INERT until you set it.

WHY IT'S SAFE TO SHIP NOW
- Flag default OFF (env WHY_TODAY_TIEBREAK unset). With it off, /still/score uses
  the SAME prompt and the SAME cache key as today — byte-for-byte unchanged. The
  app already sends today+recentThemes context, but the engine ignores it and
  excludes it from the cache key while the flag is off.
- No migration. No PROMPT_VERSION bump. (When the flag is ON, results are cached
  under a separate key that folds in the context, so turning it on/off never
  serves mismatched cached results.)

   NOTE: the existing fixtures send NO context, so even with the flag on the

4. BEHAVIOR CHECK — confirm the tiebreak actually fires when it should. Either:

5. If both pass and the feel is right, leave WHY_TODAY_TIEBREAK=on. To roll back
